# 04 - 版本路线图 (Roadmap V0-V5)

> 每个版本结束时都可 `love .` 运行，且打 git tag。
> 验收标准是 "必须全部满足" 才能打 tag。

---

## V0 — "Hello Engine"（开山版本）
**Tag:** v0.0.0
**主题:** 把 LÖVE2D 跑起来，搭建完全与象棋无关的通用脚手架

### 目标
让一个彩色分层窗口弹出来，能按键盘切换测试场景，日志在控制台滚动。证明核心链路通了。

### 范围（做什么）
- [x] LÖVE2D 项目配置 conf.lua
- [x] main.lua 入口
- [x] Core 层全部模块：
  - config.lua — 配置系统（读写 config.lua 返回的 table）
  - logger.lua — 日志（debug/info/warn/error，控制台输出）
  - event.lua — 事件总线（on/off/emit）
  - input.lua — 输入抽象（keyboard/mouse 状态，映射为 action）
  - resource.lua — 资源管理（getImage/getFont/getSound，带缓存）
  - renderer.lua — 分层渲染（5 层：background/world/effects/ui/overlay）
  - scene_manager.lua — 场景切换（fade 过渡）
  - timer.lua — 计时器 + Tween
  - utils.lua — 工具函数（深拷贝、表打印、颜色等）
  - engine.lua — 调度 load/update/draw
- [x] 两个测试场景：
  - BootScene：显示 loading 文字，1 秒后切到 DemoScene
  - DemoScene：彩色方块 + "Hello Engine" + 按 1/2/3 切层颜色 + 按 ESC 退出

### 范围外（不做什么）
- 任何象棋相关代码
- 物理引擎
- 音频播放
- UI 组件
- 外部图片资源（V0 全部用 LÖVE2D 内置图形绘制）

### 验收标准
1. `love .` 能启动窗口，标题 "Trillion Games 2D - Vale"
2. 控制台能看到日志输出（BOOT → SWITCH SCENE → FPS 等）
3. 窗口中央显示 "Hello Engine" 文字和彩色方块
4. 按 1/2/3 能改变方块颜色，按 ESC 退出
5. 能看到分层（背景暗色、世界层彩色方块、UI 层文字）
6. 所有 Core 模块 require 无报错

### 为什么先做这个
"地基"。后面所有东西都依赖这些模块。V0 把启动链路、模块加载模式、事件、场景、渲染分层全跑通，V1+ 只需专注"往框架里填游戏内容"。

## V1 — "棋盘初现"
**Tag:** v1.0.0
**主题:** 渲染中国象棋棋盘和棋子，有伪纵深感

### 目标
打开窗口看到一张完整的中国象棋棋盘，棋子在初始位置，视觉上有"木头棋盘+玉石棋子"的质感。

### 范围
- [ ] 棋盘渲染：9 条竖线 × 10 条横线、楚河汉界、九宫格斜线
- [ ] 棋盘视觉：木纹底色（用 LÖVE2D 画渐变+纹理模拟）、15° 斜视角偏移、棋盘侧面厚度（深色矩形模拟）
- [ ] 棋子渲染：32 个棋子在初始位置，圆形 + 中文文字（黑红两色），椭圆投影
- [ ] 字体加载：加载中文 TrueType 字体（思源黑体或系统中文字体）
- [ ] Camera/Viewport：固定镜头，棋盘居中，支持窗口缩放适配
- [ ] BoardScene：新场景，替代 V0 的 DemoScene
- [ ] 简单的棋盘坐标转换：屏幕坐标 ↔ 棋盘格坐标

### 范围外
- 棋子移动、选中
- 走法规则
- 动画

### 验收标准
1. 窗口显示完整棋盘（9×10 网格 + 楚河汉界 + 九宫斜线）
2. 32 个棋子在正确初始位置（帅/将居中底线，兵卒在第4/7行等）
3. 棋子下方有椭圆阴影，视觉上"浮"在棋盘上
4. 棋盘有"侧面厚度"——底部边缘有深色条，看起来像立体棋盘
5. 棋盘在窗口居中，文字清晰可辨
6. 窗口大小变化时棋盘仍然居中（如果做了）

### 为什么 V1 做这个
视觉验证最能给 solo 开发者正反馈。看到棋盘那一刻你会觉得"这是一个真游戏了"。同时验证了渲染器、资源管理、字体、坐标系统能支撑真实游戏内容。

## V2 — "落子有声"
**Tag:** v2.0.0
**主题:** 交互 + 走法 + 基础反馈

### 目标
点击棋子能选中、看到可走位置、点过去能移动；有简单音效和动画。

### 范围
- [ ] 鼠标点击选中棋子（高亮 + 发光效果）
- [ ] 显示可走点（小圆点标记合法落子位）
- [ ] 各兵种走法校验：将/帅（九宫一步）、士/仕（九宫斜走）、象/相（田字不过河）、马（日字+蹩马腿）、车（直线）、炮（直线+隔子打）、兵/卒（过河前只能前进，过河后可横走）
- [ ] 点击合法位置 → 棋子移动过去（用 tween 做缓动动画 easeOutQuad，200ms）
- [ ] 吃子：目标棋子消失 + 粒子效果（碎片飞散）+ 音效
- [ ] 回合切换：红走一步 → 黑走 → 红走...
- [ ] Engine 层新增：audio/ 模块（封装 love.audio）、anim/tween.lua（V0 timer 已预留）、particles/（简单粒子发射器）
- [ ] 简单音效：落子声（低频"哒"）、吃子声（清脆"咔"）

### 范围外
- 将军/将死判定
- AI
- 悔棋
- 菜单

### 验收标准
1. 红方先手，点一个红棋能选中（高亮）
2. 选中后显示该棋子所有合法落点
3. 点合法位置棋子平滑移动过去
4. 吃子时目标消失，有粒子+声音
5. 马能正确处理蹩马腿，炮能正确处理隔子打
6. 非法移动（比如车拐弯）不会执行
7. 红走完自动轮到黑（可手动点黑棋继续走）

### 为什么 V2 做这个
这是"从画面变游戏"的关键一步。核心交互循环（看→选→走→反馈）建立后，游戏就活了。

## V3 — "棋逢对手"
**Tag:** v3.0.0
**主题:** 完整规则 + AI 对手

### 目标
游戏可以完整下完一盘棋，能判胜负，有 AI 可以对战。

### 范围
- [ ] 将军检测：被将军时自动提示（红色闪烁）
- [ ] 将死/困毙判定：游戏结束弹窗
- [ ] 长将/长捉判和（简单版本：重复局面三次判和）
- [ ] 回合指示器（"红方走"/"黑方走"）
- [ ] 游戏结束界面（"红方胜！"/"黑方胜！"/"和棋"）
- [ ] 悔棋功能（回退一步）
- [ ] AI 对手：Minimax + Alpha-Beta 剪枝
  - 难度1：搜索深度 1（随机走合法位置）
  - 难度2：搜索深度 2
  - 难度3：搜索深度 3-4 + 简单局面评估（子力价值+位置价值）
- [ ] 人机/双人切换按钮
- [ ] 走棋历史（右边栏显示"车二平五"等简记）
- [ ] Engine 层新增：fsm/（有限状态机，用于游戏状态）、ui/ 基础组件

### 范围外
- 主菜单
- 存档
- 设置

### 验收标准
1. 一方将死对方时弹出胜负界面
2. 将军时被将军方的帅/将红色闪烁提示
3. 难度 3 的 AI 能基本走对（不送子，会吃子）
4. 悔棋能回退一步
5. 能完整下完一盘棋（不管是吃将还是困毙）
6. 回合指示正确

### 为什么 V3 做这个
V3 结束时你就有一个"能玩的完整象棋游戏"了。这是从 demo 到 game 的质变。AI 是最有技术含量的部分，单独一个版本专注做好。

## V4 — "锦上添花"
**Tag:** v4.0.0
**主题:** 打磨、菜单、音效、存档、视觉升级

### 目标
游戏看起来和玩起来像一个成品。有主菜单、有设置、有存档、音效音乐到位。

### 范围
- [ ] 主菜单 Scene：开始游戏、设置、退出
- [ ] 设置 Scene：音量（BGM/SFX）、难度、全屏、分辨率
- [ ] 存档系统：保存当前棋局、棋谱记录（engine/save/ 模块）
- [ ] 残局加载（可选：几个经典残局 preset）
- [ ] BGM：循环的古风背景音乐（音量可调）
- [ ] 视觉升级：
  - 棋子选中时的光圈/脉冲动画
  - 落子时的镜头微震（吃子时更明显）
  - 棋盘光影 shader（径向光从中心向外渐暗）
  - 胜利时的粒子烟花
- [ ] 引擎模块：engine/ui/ 完善（Button/Panel/Label/Slider）
- [ ] 简单的 string i18n 预留（中文 key → 中文/English）

### 范围外
- Steam SDK
- Android 打包
- 联机

### 验收标准
1. 打开游戏先到主菜单，点击"开始"进入棋盘
2. 设置菜单能改音量/全屏/难度并生效
3. 能中途保存棋局，退出重进后可继续
4. BGM 循环播放，SFX 音效清晰
5. 视觉反馈到位（选中脉冲、吃子震动、胜利烟花）
6. 整体观感不像个人 demo 了

### 为什么 V4 做这个
玩家第一眼看到的是菜单、听到的是音乐、感受到的是动画。V3 是"能玩"，V4 是"想玩"。

## V5 — "整装待发"
**Tag:** v5.0.0
**主题:** 打包发布（Steam/桌面/APK）

### 目标
能构建出可分发的包，上架路径清晰。

### 范围
- [ ] 游戏图标、名称、版本号配置
- [ ] 桌面打包：
  - Windows: .exe（通过 love-release 或手动打包，附 love.dll）
  - Linux: .AppImage
  - macOS: .app（注意公证问题）
- [ ] Steam：
  - 注册 Steam 开发者（$100）
  - 下载 Steamworks SDK
  - 用 luasteam 或 Greenworks 集成（Steam  overlay 至少能打开）
  - Steam 成就（可选，比如"首次将死""100局胜利"等）
  - Steam 云存档（可选，用 love.filesystem + Steam Remote Storage）
  - 上传 Steam Depot
- [ ] APK：
  - 使用 love-android (https://github.com/love2d/love-android)
  - 适配触摸：大棋子、点击区域放大
  - 返回键处理（退出确认对话框）
  - 签名打包
- [ ] 性能优化：
  - 棋子使用 Canvas 预渲染，不每帧重绘
  - AI 计算移到协程（coroutine）避免卡帧
- [ ] 构建脚本自动化（build.sh / Makefile）
- [ ] 发布检查清单（README 补全、键位说明、截图）

### 范围外
- iOS（后续看需求）
- WebAssembly（后续看需求）
- 联机对战（下一阶段）

### 验收标准
1. `make windows` 能产出在干净 Windows 上双击运行的 .exe
2. `make linux` 能产出 .AppImage
3. `make apk` 能产出 debug APK 并在手机安装运行
4. Steam 包在 Steam 客户端内能启动（至少 DRM-free 版能跑）
5. Android 触摸操作正常
6. 帧率在所有目标平台稳定 60fps

### 为什么 V5 最后做
打包发布是最终一步。先保证游戏好玩再打包，不然包再精美游戏不好玩也没人买。同时打包会暴露平台相关 bug，放到最后集中处理。
